**Shared Components (frontend/src/components)**

- `Navbar.tsx`: Main site navigation bar. Hides on `/admin` and `/vendor` routes. Contains top strip, main nav links, login/register actions, and mobile menu placeholder.
- `Footer.tsx`: Site footer with quick links, contact/support, supported-by info. Hidden on `/admin` and `/vendor` routes.
- `AuthProvider.tsx`: Context/provider for authentication state. Wraps app to provide user session, role checks, login/logout utilities.
- `RoleGuard.tsx`: Higher-order component or wrapper to restrict access to routes/components based on user role (e.g., officer vs vendor).
- `AdminPageContent.tsx`: Renders subpages inside admin dashboard depending on `page` query (tenders, bids, disputes, reports, profile, notifications).
- `OfficerSidebar.tsx`: Sidebar navigation component used in admin views (links to admin sections and actions).

- `KYCStatusBadge.tsx`: Small badge component that renders KYC status (approved, pending, under-review) with colors.
- `KYCStatusStepper.tsx`: Multi-step UI used in KYC flow to show progress through verification steps.
- `AuditTimeline.tsx`: Timeline component to render audit events and ledger timeline for tenders or actions.
- `SystemHealthPanel.tsx`: Component showing system health boxes (blockchain node, IPFS, KYC, Email) — used in admin dashboard.
- `IPFSHashPill.tsx`: Small UI pill to show IPFS hash with copy/preview actions.
- `ImmutableBadge.tsx`: Visual badge indicating immutability or blockchain anchoring of a record.
- `EncryptedBidBadge.tsx`: Badge indicating a bid is encrypted/locked until opening.
- `WalletConnectButton.tsx`: Button component that triggers wallet connection flow for signing transactions.
- `BlockchainTxModal.tsx`: Modal UI to show blockchain transaction progress, confirmation, and hash after publishing a tender.
- `BidCountdown.tsx`: Countdown timer component for bid closing time, used on tender detail pages.
- `WinnerBanner.tsx`: Banner displayed when a tender has an awarded winner.
- `WeightSlider.tsx`: Small interactive slider used for weighting criteria or scoring in tender evaluation.

**UI Primitives (frontend/src/components/ui)**
- `button.tsx`: Reusable styled Button wrapper used across the app (accepts `variant`, `size` props).
- `card.tsx`: Card primitives (`Card`, `CardHeader`, `CardContent`, `CardTitle`, `CardDescription`) used for layout blocks.
- `input.tsx`: Form input primitives (text inputs, selects) with consistent styling.
- `badge.tsx`: Small badge UI used for statuses and inline labels.

Notes & guidance for maintainers
- For pages that render data (tenders, dashboard, admin), look into `frontend/src/data/*.json` for mock data used in local dev: `tenders.json`, `vendors.json`, `tenderBids.json`, `systemHealth.json`, etc.
- Authentication and role gating are implemented in `AuthProvider.tsx` and `RoleGuard.tsx` — follow these when adding protected routes.
- Admin actions (create tender, publish) interact with `BlockchainTxModal` and likely call backend APIs or web3 providers via `WalletConnectButton`.
- UI primitives under `components/ui` are the place to change design tokens and shared control styles.
